# Assessment Requirement Traceability

This checklist maps each required item from the Valsoft brief to code and evidence in this repository.

## Section 3.1 - Required Workflow Steps

| Requirement | Implementation | Evidence Artifact |
|---|---|---|
| Step 1: Ingestion auto-start on new message | `ingestion/webhook_api.py` (`POST /intake`) | `docs/demo_runbook.md` webhook demo steps |
| Step 2: Classification (category, priority, confidence) | `workflow/nodes.py` -> `classify_node` | `output/submission_records.json` |
| Step 3: Enrichment (core issue, identifiers, urgency) | `workflow/nodes.py` -> `enrich_node` | `output/submission_records.json` |
| Step 4: Routing to destination queue + fallback | `workflow/nodes.py` -> `route_node` | `proposed_queue` and `destination_queue` fields |
| Step 5: Structured JSON output + human summary | `workflow/nodes.py` -> `output_node`; `scripts/generate_submission_artifacts.py` | `output/submission_records.json` and `output/submission_summary.md` |
| Step 6: Human escalation when confidence < 70% or escalation criteria | `workflow/nodes.py` escalation rules; `config/settings.py` thresholds | `escalation_flag`, `escalation_rules_triggered`, `destination_queue=Human Review` |

## Section 4 - Deliverables

| Deliverable | Implementation / Location |
|---|---|
| 4.1 Working workflow | `app.py` (UI), `main.py` (CLI), `ingestion/webhook_api.py` (auto-ingestion path) |
| 4.2 Structured output file | `output/submission_records.json` (generated by `scripts/generate_submission_artifacts.py`) |

## Evaluation Criteria Coverage

| Dimension | Repo Evidence |
|---|---|
